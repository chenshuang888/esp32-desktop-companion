# ESP32-S3 LVGL 项目开发日志

## 项目概述

基于 ESP32-S3 的 LVGL GUI 应用，使用 ST7789 LCD 显示屏和 FT5x06 触摸屏，实现多页面导航和时间调节功能。

**技术栈：**
- ESP-IDF v5.4.3
- LVGL v9.5.0
- FreeRTOS
- ST7789 LCD (240x320, SPI, RGB565)
- FT5x06 触摸 (I2C)

---

### 阶段三：页面路由框架设计

**任务：** 实现多页面导航系统

**设计思路：**
- 参考 Web 开发的路由概念
- 页面生命周期管理：create / destroy / update
- 页面注册机制
- 页面切换 API

**架构设计：**

```
framework/
  └── page_router.h/c    # 路由框架核心

app/pages/
  ├── page_time.h/c      # 时间调节页面
  └── page_menu.h/c      # 菜单页面
```

**页面生命周期回调：**
```c
typedef struct {
    lv_obj_t *(*create)(void);  // 创建页面
    void (*destroy)(void);       // 销毁页面
    void (*update)(void);        // 更新页面（在 UI 线程中调用）
} page_callbacks_t;
```

**遇到的问题：**
1. **头文件包含路径错误**：使用了 `#include "framework/page_router.h"`
2. 解决方案：改为 `#include "page_router.h"`，因为 CMakeLists.txt 的 REQUIRES 会自动添加 include 路径

**经验教训：**
- ESP-IDF 组件系统会自动处理 include 路径
- 不要使用相对路径包含其他组件的头文件

### 阶段四：代码架构重构（HTML/CSS/JS 分离模式）

**任务：** 将页面代码按照 Web 开发模式重构

**设计理念：**
- **CSS 部分**：`init_styles()` - 初始化所有样式
- **HTML 部分**：`create_xxx_card()` - 创建 UI 布局
- **JavaScript 部分**：`on_xxx_clicked()` - 事件处理
- **入口函数**：`page_init()` - 组装所有部分

**代码结构示例：**
```c
/* CSS - 样式定义 */
static void init_styles(void) {
    lv_style_init(&s_ui.style_card);
    lv_style_set_bg_color(&s_ui.style_card, lv_color_hex(0x2C3E50));
    // ...
}

/* HTML - 布局创建 */
static void create_time_card(void) {
    lv_obj_t *card = lv_obj_create(s_ui.screen);
    lv_obj_add_style(card, &s_ui.style_card, 0);
    // ...
}

/* JavaScript - 事件响应 */
static void on_hour_up_clicked(lv_event_t *e) {
    adjust_time(1, 0, 0);
}

/* 入口 */
static void page_init(void) {
    init_styles();
    create_time_card();
    create_date_card();
    bind_events();
}
```

**优势：**
- 代码结构清晰，职责分离
- 易于维护和修改
- 符合前端开发习惯

**遇到的问题：**
1. **函数声明顺序错误**：JavaScript 部分的函数在 HTML 部分之前使用
2. 解决方案：调整代码顺序，JavaScript 部分放在 HTML 部分之前

## 核心 Bug 修复历程：页面切换导致看门狗超时

### Bug 现象

**初始症状：**
- 第一次切换：时间页面 → 菜单页面（成功）
- 第二次切换：菜单页面 → 时间页面（成功）
- 第三次切换：时间页面 → 菜单页面（**卡死，看门狗超时**）

**堆栈信息：**
```
E (29260) task_wdt: Task watchdog got triggered
Backtrace: ... -> lv_draw_sw_box_shadow -> ...
```

卡死在绘制阴影效果时。

---

### 修复尝试 #1：移除全局静态 style 对象

**问题分析：**
- 原代码使用全局静态 `lv_style_t` 变量
- 有 `s_styles_initialized` 标志防止重复初始化
- style 对象在页面销毁后仍然存在，内部可能保留了指向已删除 UI 对象的指针

**修复方案：**
- 将 style 对象移到 `page_time_ui_t` 结构体中
- 移除 `s_styles_initialized` 标志
- 每次创建页面时重新初始化 style

**代码变更：**
```c
typedef struct {
    lv_obj_t *screen;
    lv_obj_t *time_label;
    // ... 其他 UI 对象
    
    // 样式对象移到这里
    lv_style_t style_card;
    lv_style_t style_time_label;
    lv_style_t style_date_label;
    lv_style_t style_btn;
    lv_style_t style_btn_pressed;
} page_time_ui_t;
```

**修复结果：**
- ❌ 失败，问题依然存在
- 原因：destroy 函数没有正确清理 style 对象

---

### 修复尝试 #2：使用 memset 清零结构体

**修复方案：**
```c
void page_time_destroy(void) {
    ESP_LOGI(TAG, "Destroying time page");
    memset(&s_ui, 0, sizeof(s_ui));
}
```

**修复结果：**
- ❌ 失败，问题变得更严重
- 第二次切换就卡死了（比之前更早）

**问题分析：**
- `memset` 直接清零了 style 对象的内存
- 破坏了 LVGL style 对象的内部状态
- LVGL 的 style 对象有内部链表、引用计数等复杂结构

---

### 修复尝试 #3：先 lv_style_reset 再 memset

**修复方案：**
```c
void page_time_destroy(void) {
    ESP_LOGI(TAG, "Destroying time page");
    
    // 先重置 style 对象
    lv_style_reset(&s_ui.style_card);
    lv_style_reset(&s_ui.style_time_label);
    lv_style_reset(&s_ui.style_date_label);
    lv_style_reset(&s_ui.style_btn);
    lv_style_reset(&s_ui.style_btn_pressed);
    
    // 再清零结构体
    memset(&s_ui, 0, sizeof(s_ui));
}
```

**修复结果：**
- ❌ 失败，第四次切换时卡死

**新的堆栈信息：**
```
Backtrace: ... -> lv_array_resize -> lv_array_push_back -> lv_event_add
-> lv_label_create -> create_button -> create_time_card
```

卡死在创建 label 时的动态数组扩容。

**问题分析：**
1. `lv_style_reset()` 正确清理了 style 内部状态
2. 但 `memset()` 又把这些状态清零了
3. style 对象处于不一致状态：LVGL 认为它已被清理，但内存状态是全零
4. 下次 `lv_style_init()` 时，LVGL 检测到异常状态，跳过某些初始化
5. 后续使用时访问未正确初始化的内存，导致死循环

---

### 修复尝试 #4：正确的清理顺序（最终方案）✅

**关键认识：**
- LVGL 的对象（style、obj 等）有内部状态管理
- 必须用 LVGL 提供的 API 操作，不能用 `memset` 暴力清零
- 类似 C++ 对象，需要调用构造/析构函数

**正确的清理顺序：**
1. 先删除 UI 对象（解除 style 引用）
2. 再调用 `lv_style_reset()`（清理 style 内部状态）
3. 不要 `memset` style 对象！

**最终代码：**
```c
void page_time_destroy(void)
{
    ESP_LOGI(TAG, "Destroying time page");

    // 1. 删除 screen 会自动删除所有子对象，并解除 style 引用
    if (s_ui.screen) {
        lv_obj_del(s_ui.screen);
        s_ui.screen = NULL;
    }

    // 2. 重置所有 style 对象，清理 LVGL 内部状态
    lv_style_reset(&s_ui.style_card);
    lv_style_reset(&s_ui.style_time_label);
    lv_style_reset(&s_ui.style_date_label);
    lv_style_reset(&s_ui.style_btn);
    lv_style_reset(&s_ui.style_btn_pressed);

    // 3. 清零 UI 对象指针（不包括 style 对象）
    s_ui.time_label = NULL;
    s_ui.date_label = NULL;
    s_ui.hour_up_btn = NULL;
    s_ui.hour_down_btn = NULL;
    s_ui.min_up_btn = NULL;
    s_ui.min_down_btn = NULL;
    s_ui.year_up_btn = NULL;
    s_ui.year_down_btn = NULL;
    s_ui.mon_up_btn = NULL;
    s_ui.mon_down_btn = NULL;
    s_ui.day_up_btn = NULL;
    s_ui.day_down_btn = NULL;
    s_ui.menu_btn = NULL;
}
```

**修复结果：**
- ✅ 成功！页面可以无限次切换，不再卡死

**测试结果：**
```
I (12144) page_time: Destroying time page
I (12147) page_router: Switching to page 1
I (12147) page_menu: Creating menu page
I (12153) page_router: Switched to page 1
I (15850) page_menu: Destroying menu page
I (15852) page_router: Switching to page 0
I (15852) page_time: Creating time page
I (15905) page_router: Switched to page 0
... (多次切换，全部成功)
```

## 核心经验总结

### 1. LVGL 对象生命周期管理

**关键原则：**
- LVGL 的对象（`lv_obj_t`、`lv_style_t` 等）都有内部状态
- 必须使用 LVGL 提供的 API 进行初始化和清理
- 不能用 `memset` 等方式暴力清零

**正确的 style 对象生命周期：**
```c
// 初始化
lv_style_init(&style);
lv_style_set_xxx(&style, ...);

// 使用
lv_obj_add_style(obj, &style, 0);

// 清理（在删除使用该 style 的对象之后）
lv_style_reset(&style);
```

**错误示例：**
```c
// ❌ 错误：直接 memset 清零
lv_style_reset(&style);
memset(&style, 0, sizeof(style));  // 破坏内部状态！

// ❌ 错误：在对象还在使用时清理 style
lv_style_reset(&style);
lv_obj_del(obj);  // 顺序反了！
```

### 2. 结构体设计原则

**当结构体包含需要特殊处理的对象时：**
- 不要对整个结构体使用 `memset`
- 分别清理每个成员
- 或者只清理简单类型成员

**示例：**
```c
typedef struct {
    // 简单类型，可以清零
    lv_obj_t *screen;
    lv_obj_t *label;
    
    // 复杂对象，需要特殊处理
    lv_style_t style;
} my_ui_t;

// 清理时
void cleanup(my_ui_t *ui) {
    // 先清理复杂对象
    lv_style_reset(&ui->style);
    
    // 再清理简单类型
    ui->screen = NULL;
    ui->label = NULL;
    
    // ❌ 不要这样做：
    // memset(ui, 0, sizeof(*ui));  // 会破坏 style！
}
```

### 3. 调试复杂 Bug 的方法

**我们的调试过程：**
1. 观察现象：第几次切换卡死
2. 查看堆栈：卡在哪个函数
3. 分析原因：为什么会卡在那里
4. 提出假设：可能是什么问题
5. 验证假设：修改代码测试
6. 如果失败：重新分析堆栈，寻找新线索

**关键线索的变化：**
- 第一次：卡在 `lv_draw_sw_box_shadow`（绘制阴影）
- 第二次：卡在 `lv_array_resize`（动态数组扩容）
- 第二次的堆栈更有价值，指向了内存管理问题

**经验：**
- 不要只看表面现象，要深入分析堆栈
- 每次修改后的新现象都是重要线索
- 问题变得更严重不一定是坏事，可能暴露了更深层的问题

### 4. 代码架构设计

**HTML/CSS/JS 分离模式的优势：**
- 代码结构清晰，易于理解
- 职责分离，便于维护
- 符合前端开发习惯，降低学习成本

**适用场景：**
- 嵌入式 GUI 开发
- 需要频繁修改 UI 的项目
- 团队协作开发

### 5. ESP-IDF 开发注意事项

**字体配置：**

- 在 `sdkconfig.defaults` 中声明
- 必要时手动编辑 `sdkconfig`

**组件系统：**

- 不要使用相对路径包含其他组件的头文件

- 避免使用 UTF-16 或 GBK
